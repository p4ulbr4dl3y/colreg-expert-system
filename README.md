# Экспертная система предупреждения столкновений судов

Продуктовое решение для оценки навигационных рисков и выработки рекомендаций по расхождению судов в соответствии с Международными правилами предупреждения столкновений судов в море (МППСС-72).

Система анализирует координаты и параметры движения собственного судна и окружающих судов-целей, оценивает риски столкновения, строит карту опасных секторов курса и предлагает безопасные траектории маневрирования с учетом маневренных характеристик судна.

## Основные возможности

Система предоставляет следующий набор функций:
- многоцелевой анализ обстановки: одновременный расчет рисков для произвольного количества окружающих целей;
- расчет геометрических параметров сближения: вычисление дистанции кратчайшего сближения (CPA) и времени до него (TCPA);
- определение взаимного статуса и приоритета судов: классификация ситуаций по секторам обгона, встречного сближения и пересечения курсов с учетом приоритета типов судов согласно МППСС-72;
- динамическая адаптация зоны безопасности: масштабирование защитного радиуса при обнаружении целей в критической близости;
- учет ограничений авторулевого: автоматическая блокировка поворотов на левый борт в ситуациях встречного сближения и пересечения курсов согласно правилам 14, 15 и 17;
- проверка физических ограничений маневрирования: проверка возможности завершения маневра до наступления момента кратчайшего сближения на основе радиуса циркуляции и скорости судна;
- поддержка работы в условиях ограниченной видимости: применение логики правила 19 с автоматической подстройкой ограничений на развороты в зависимости от положения целей относительно траверза.

## Архитектура проекта

Кодовая база разделена на следующие модули:
- `src/models.py`: содержит структуры данных для судов, окружающей среды, индивидуальных и общих решений;
- `src/geometry.py`: реализует векторную математику для расчета траекторий сближения, секторов опасных курсов и параметров поворота;
- `src/rules.py`: содержит правила приоритета типов судов (правило 18) и расхождения парусных судов в зависимости от галса и ветра (правило 12);
- `src/engine.py`: координирует логический вывод экспертной системы, объединяя требования всех правил и строя итоговую рекомендацию по маневрированию;
- `main.py`: предоставляет консольный интерфейс для ручного тестирования и запуска демонстрационных сценариев;
- `scripts/mqtt_node.py`: реализует асинхронный сетевой узел, принимающий навигационные данные и публикующий результаты через протокол MQTT;
- `scripts/mqtt_simulate.py`: отправляет тестовый навигационный запрос в брокер сообщений для верификации работы сетевого узла.

## Установка и требования

Для работы требуется установленный интерпретатор Python версии 3.12 или выше. Рекомендуется использовать менеджер пакетов `uv`.

Шаги по установке:
1. Клонируйте репозиторий:
   ```bash
   git clone git@github.com:p4ulbr4dl3y/expert-system.git
   cd expert-system
   ```
2. Создайте виртуальное окружение и установите зависимости:
   ```bash
   uv sync
   ```

## Запуск демонстрационного интерфейса

Запуск интерактивного консольного меню выполняется командой:
```bash
uv run python main.py
```
В меню доступны следующие действия:
- 1: запуск стандартных навигационных сценариев расхождения (включая многоцелевой конфликт и проверку инерционных ограничений);
- 2: ввод параметров собственного судна и целей вручную для проведения расчетов;
- 3: выход из программы.

## Работа по протоколу MQTT

Экспертный узел может быть интегрирован в судовую сеть. Он обменивается сообщениями по следующим топикам:
- `colreg/expert/command`: топик для входящих команд с параметрами судов;
- `colreg/expert/result`: топик для публикации выработанных решений.

Для проверки работы выполните следующие шаги:
1. Запустите MQTT-брокер на локальном хосте (порт 1883).
2. Запустите сетевой узел экспертной системы:
   ```bash
   uv run scripts/mqtt_node.py
   ```
3. В отдельном терминале отправьте тестовый запрос через симулятор:
   ```bash
   uv run scripts/mqtt_simulate.py
   ```

## Тестирование

Запуск набора автоматических тестов, проверяющих расхождение в различных ситуациях:
```bash
uv run python -m unittest discover tests
```
